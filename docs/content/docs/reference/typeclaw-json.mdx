## Top-level fields

| Field                   | Type                                            | Reload                                       | Concept                                                                                  |
| ----------------------- | ----------------------------------------------- | -------------------------------------------- | ---------------------------------------------------------------------------------------- |
| `models`                | `Record<string, ref \| ref[] \| ProfileObject>` | live                                         | model profile → ref, fallback chain, or object with a per-profile `thinkingLevel`        |
| `port`                  | `number`                                        | restart-required                             | preferred host port; CLI falls back to ephemeral on conflict                             |
| `mounts`                | `Mount[]`                                       | restart-required                             | host files or directories exposed inside the container                                   |
| `plugins`               | `string[]`                                      | restart-required                             | plugin module specifiers                                                                 |
| `alias`                 | `string[]`                                      | live                                         | additional names the agent answers to in channels                                        |
| `channels`              | `Record<adapter, AdapterConfig>`                | live                                         | per-adapter config                                                                       |
| `portForward`           | `{ allow, deny? }`                              | restart-required                             | auto port-forward allow/deny lists; default `{ allow: '*' }`                             |
| `network`               | `NetworkPolicy`                                 | restart-required                             | egress filtering                                                                         |
| `sandbox`               | `SandboxPolicy`                                 | restart-required                             | per-tool bwrap sandbox options                                                           |
| `tunnels`               | `Tunnel[]`                                      | restart-required                             | public URLs (see [/concepts/architecture](/docs/concepts/architecture))                  |
| `roles`                 | `Record<name, Role>`                            | `match` live, `permissions` restart-required | permission roles (see [/concepts/permissions-model](/docs/concepts/permissions-model))   |
| `docker.file`           | `DockerFileOptions`                             | rebuild via `start`                          | Dockerfile feature toggles + append lines                                                |
| `docker.runArgs.append` | `string[]`                                      | rebuild via `start`                          | extra `docker run` flags                                                                 |
| `git.ignore.append`     | `string[]`                                      | rebuild via `start`                          | extra `.gitignore` lines                                                                 |
| `updateCheck`           | `{ enabled }`                                   | host-only                                    | toggle the once-a-day "newer typeclaw available" CLI notice; default `{ enabled: true }` |
| `branding`              | `boolean`                                       | live                                         | disclose the TypeClaw runtime in the system prompt; default `true`                       |

Plugin-owned config blocks live at the top level under their plugin name and are validated by the plugin's own `configSchema`.

The `updateCheck` notice is host-stage only — the container never reads it. Every CLI command does a cheap disk read of a cached "latest" version under `~/.typeclaw/` and prints a single line to stderr when a newer release exists; a detached, throttled (once per 24h) background process refreshes that cache off the hot path. The check never blocks or fails a command. Disable it persistently with `"updateCheck": { "enabled": false }`, or per-invocation with the `TYPECLAW_NO_UPDATE_CHECK` / `NO_UPDATE_NOTIFIER` / `CI` env vars.

`branding` controls whether the system prompt discloses the runtime. With the default `true`, the prompt tells the agent it runs "inside TypeClaw" and includes a `## Runtime` block stamping the TypeClaw version. Set `"branding": false` to strip both: the base prompt opens generically ("a general-purpose AI agent" / "an AI agent" for slim sessions) and the runtime-version block is omitted — including on the subagent prompt path. It is read fresh per session, so it applies on `typeclaw reload` without a restart. Functional identifiers the agent still needs — the `typeclaw.json` filename and the `typeclaw-render-pdf` / `typeclaw-troubleshooting` skill names — are unaffected; they are system-defined tokens, not runtime disclosure.

## `models`

```json
{
  "models": {
    "default": "openai/gpt-5.4-nano",
    "fast": "openai/gpt-5.4-mini",
    "deep": ["anthropic/claude-opus-4-8", "fireworks/accounts/fireworks/routers/kimi-k2p6-turbo"],
    "vision": "openai/gpt-5.4"
  }
}
```

`default` is required. Other profile names are conventional but not enforced. Each value is either a single `<provider>/<model>` ref or a non-empty array forming a fallback chain.

The chain advances on failure, but **how** it advances depends on the turn driver. Cron and other batch turns replay the prompt against the next ref on **any** error. Interactive turns (TUI, channels, subagents) only fail over when the active ref is **throttled or overloaded** (`server_is_overloaded`, `429`, `503`, rate-limit) — a one-off fault like a malformed response or a billing/auth error surfaces on the active ref instead of silently switching providers. Interactive failover also will not switch mid-turn once the model has produced output or run a tool, so a partially-completed turn is never duplicated. A repeatedly-throttled ref is skipped for a short cooldown (a per-process circuit breaker) so every turn does not rediscover the same saturated provider. A **single-ref** profile has no fallback to advance to: a throttled turn fails fast with a provider notice rather than stalling on retries.

### Per-profile reasoning effort

A profile value may also be an object that pairs its model(s) with a `thinkingLevel`:

```json
{
  "models": {
    "default": { "model": "openai/gpt-5.4-nano", "thinkingLevel": "high" },
    "fast": { "model": "openai/gpt-5.4-mini", "thinkingLevel": "off" },
    "deep": {
      "models": ["anthropic/claude-opus-4-8", "fireworks/accounts/fireworks/routers/kimi-k2p6-turbo"],
      "thinkingLevel": "xhigh"
    },
    "vision": "openai/gpt-5.4"
  }
}
```

`thinkingLevel` (`off` through `xhigh`) sets the reasoning effort for sessions running under that profile. There is no global `thinkingLevel` field — the **`default` profile's** `thinkingLevel` is the de-facto global default, exactly as `models.default` is the default model.

A session resolves its effort with the precedence **the profile's own `thinkingLevel` → the `default` profile's `thinkingLevel` → the SDK default**. So `vision` above (a bare ref with no level) inherits whatever `default` declares; if `default` has no level either, the SDK default (`medium`) applies. The value is clamped to each model's capabilities at session creation (`xhigh` is OpenAI-family-only and clamps down to `high` elsewhere). An attention-escalation cue in a turn (frustration or an explicit "do it properly" / "ultrathink") still bumps that single turn to maximum effort regardless of this setting.

Two well-known profiles ship with a built-in `thinkingLevel`: **`fast` defaults to `low`** and **`deep` defaults to `high`**. These are materialized into the profile at config-load time, so they show up in `typeclaw model list` and win over a lower `default` level (a materialized `deep: high` still thinks hard even if you lower the global default). `default` and `vision` carry no built-in level — they fall through to the `default` profile's level, then the SDK default. Setting an explicit `thinkingLevel` on `fast` or `deep` (including `off`) overrides the built-in.

The object form accepts `model` (a single ref) **or** `models` (a single ref or a fallback-chain array) — exactly one — plus the optional `thinkingLevel`. Bare string/array profile values keep working unchanged (no migration needed); a profile only becomes an object when you attach a `thinkingLevel`. Live-reloadable: a change lands on the next session without a container restart.

Set it from the CLI with `typeclaw model set <profile> <ref> --thinking <level>` (targets that profile), or `typeclaw model thinking <level>` for the `default` profile's level. `--thinking default` clears a profile's override.

## `network`

```json
{
  "network": {
    "blockInternal": true,
    "autoAllowResolvers": true,
    "allow": []
  }
}
```

| Field                | Default | Effect                                                                                                                                                             |
| -------------------- | ------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| `blockInternal`      | `true`  | DROP traffic to RFC1918, link-local (incl. cloud IMDS at `169.254.169.254`), CGNAT, multicast/reserved, and IPv6 ULA/link-local/multicast. Loopback always allowed |
| `autoAllowResolvers` | `true`  | Narrowly carve out port 53 to nameservers in `/etc/resolv.conf` so VPC-internal DNS keeps working                                                                  |
| `allow`              | `[]`    | Explicit IPv4 CIDRs that punch through wholesale                                                                                                                   |

## `sandbox`

```json
{
  "sandbox": {
    "realProc": false,
    "writablePaths": [".metabase-cli", "workspace/cache"],
    "symlinks": [{ "from": "~/.metabase-cli", "to": "workspace/.metabase-cli" }]
  }
}
```

| Field           | Default | Effect                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                |
| --------------- | ------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `realProc`      | `false` | Opt into the stricter `real-proc` `/proc` strategy: mount a fresh, PID-namespace-scoped `/proc` in the per-tool sandbox for full PID isolation. Needs `CAP_SYS_ADMIN`, so setting `true` grants the container that cap. **Not required for package CLIs** (`bunx`, `bun run <pkg-bin>`) — the default `proc-bind` strategy already gives them a working `/proc` with no cap by `--ro-bind`ing the container's real procfs under a child user namespace that blocks `/proc/<agent>/environ` leaks. On runtimes where the `real-proc` mount is a no-op (OrbStack, rootless Docker, gVisor, Docker Desktop ECI, AppArmor hosts) the sandbox auto-falls-back to `proc-bind` regardless. See [/internals/sandbox](/docs/internals/sandbox)                                                                                                                                 |
| `writablePaths` | `[]`    | Extra **agent-root-relative** directories made writable for confined roles inside the per-tool bwrap sandbox, on top of the built-in zones (`workspace/`, `public/`, `mounts/`, `.git`). Use it when a low-trust role runs a CLI that insists on writing a fixed config dir under the agent folder. Entries must be relative (absolute paths rejected), contain no `..` or null bytes, exist as a real directory (not a file or symlink), resolve inside the agent folder, and **not** land on a security-sensitive root (`.git`, `.env`, `secrets.json`, `auth.json`, `sessions`, `memory`, `.typeclaw`, `node_modules`). Invalid or non-existent entries are silently skipped, never fatal. `trusted`/`owner` also run in bwrap, but retain an RW agent-root bind with canonical credential paths masked afterward, so they do not use this confined carve-out list |
| `symlinks`      | `[]`    | Array of `{ from, to }` that **creates a symlink and makes its target writable** in one entry. `from` is the symlink location — an absolute container path (e.g. `/root/.foo`) or a `~/`-prefixed path — and `to` is an agent-root-relative directory automatically added to `writablePaths`. The entrypoint creates the link at the real container `$HOME` for runtime-owned processes; model-driven bash creates it inside the bwrap `$HOME` (`/tmp`) for every role. Confined roles receive only links whose targets survived writable-path validation; `trusted`/`owner` retain every validated configured link. The entrypoint refuses to overwrite an existing non-symlink. `from` may not be `/`, point under `/agent`, or target the kernel/virtual roots `/proc`, `/sys`, `/dev`, `/run`. See [/internals/sandbox](/docs/internals/sandbox)                  |

All model-driven bash calls clear the inherited environment. `trusted`/`owner` retain normal agent-root writes, while `.env`, `secrets.json`, historical root `auth.json`, and the runtime-owned GWS, agent-messenger, and persisted CLI-auth directories stay masked. Reusable GWS, agent-messenger, Codex, and Claude profiles are never mounted into model-driven children; standalone local Git may receive only a generated name/email config. See [Environment variables](/docs/reference/env-vars#model-driven-bash-environment) and [/internals/sandbox](/docs/internals/sandbox).

## `mounts`

```json
{
  "mounts": [
    {
      "name": "downloads",
      "path": "~/Downloads",
      "readOnly": true,
      "description": "files to inspect"
    }
  ]
}
```

Each mount maps a host **file or directory** to `/agent/mounts/<name>` inside the container. `path` accepts absolute paths, relative paths resolved from the agent folder, and `~`/`~/...`. `readOnly` defaults to `false`; use `true` for reference data the agent should inspect but not edit. **Do not mount SSH private keys or other reusable credentials:** mounted content is part of the agent filesystem surface, not a credential broker. Use a host-side authenticated workflow or a dedicated scoped integration instead. Sockets, FIFOs, and devices are rejected. Mount changes are restart-required because Docker bind mounts are fixed when the container starts. Note that a single-file bind tracks the file's inode, so replacing the host file via write-temp-then-rename (most editors, some key-rotation tools) is not seen by the container until the next restart; mount the parent directory if you need live updates for non-secret reference data.

The CLI can manage this block:

```sh
typeclaw mount add downloads ~/Downloads --read-only
typeclaw mount list
typeclaw mount remove downloads
```

## `docker.file`

| Toggle        | Default  | Effect                                                                                                                 |
| ------------- | -------- | ---------------------------------------------------------------------------------------------------------------------- |
| `gh`          | `true`   | install GitHub CLI                                                                                                     |
| `python`      | `true`   | install Python 3 + uv                                                                                                  |
| `tmux`        | `true`   | install tmux                                                                                                           |
| `ffmpeg`      | `false`  | install ffmpeg                                                                                                         |
| `cjkFonts`    | `'auto'` | install `fonts-noto-cjk` so Chromium renders CJK glyphs; `'auto'` installs only when the host locale is CJK (ja/ko/zh) |
| `cloudflared` | `false`  | install cloudflared for `cloudflare-quick` tunnels (`tunnel add` / `channel add github` flip this on automatically)    |
| `xvfb`        | `true`   | install Xvfb for headed Chrome                                                                                         |
| `claudeCode`  | `false`  | install Anthropic's Claude Code CLI                                                                                    |
| `codexCli`    | `false`  | install OpenAI's Codex CLI (`@openai/codex` via npm)                                                                   |
| `append`      | `[]`     | extra Dockerfile lines appended verbatim                                                                               |

Boolean toggles can also take a version string (`"gh": "2.40.0"`) to pin via `apt-get install pkg=<version>`. `claudeCode` and `codexCli` are boolean-only. `cjkFonts` accepts `true`/`false`/`'auto'`: `'auto'` (the default) resolves at `typeclaw start` from the host locale, while an explicit boolean forces the decision.

## Hot reload

```sh
typeclaw reload
```

Re-reads `typeclaw.json` and applies hot-reloadable fields. Boot-only fields (`port`, `mounts`, `plugins`, `portForward`, `network`, `tunnels`) are captured once at container start; reload reports which changes require a restart.

## Auto-migration

One migration step still runs on load: if `typeclaw.json` contains a `channels.github.eventAllowlist` that was seeded by `typeclaw init`, it is dropped and the file is rewritten with a descriptive commit subject. All other legacy shapes (`dockerfile`, `gitignore`, `model`, `channels.<adapter>.allow[]`) are no longer migrated — use the current field names directly.
