TypeClaw's bundled `memory` plugin turns lived experience into reusable knowledge. No manual prompt engineering. No curated example library. Just three subagents working on different cadences.

## The loop

### 1. Observe

After every idle turn, the `memory-logger` subagent reads the session transcript and appends notable fragments to `memory/yyyy-MM-dd.md`. Cheap, frequent, lossy by design — the goal is to capture the texture of the day, not the literal log.

### 2. Dream

On a cron schedule (default 4am), the `dreaming` subagent reads the day's fragments and:

- Consolidates them into `MEMORY.md` — the agent's long-term memory, loaded into every prompt
- When it spots a repeatable procedure, writes a new **muscle-memory skill** at `memory/skills/<name>/SKILL.md`

Every dream is committed with a one-line summary, e.g.:

```
dream: 3 fragments + new skill 'pr-review' 🔮
```

So growth is auditable. You can `git log memory/` to see exactly what the agent learned and when.

### 3. Apply

The next prompt sees the updated `MEMORY.md`. Muscle-memory skills sit alongside bundled and user-installed ones, loaded on demand by the `skill` tool. The agent doesn't have to remember to use them — they're just there, with names and descriptions tuned for retrieval.

## What gets remembered

The dreaming subagent has a default prompt that nudges it toward:

- Procedures the agent ran more than once
- Mistakes the agent made (and how to avoid them)
- People-and-team facts (preferences, ownership, conventions)
- Tools and commands that worked well for specific tasks

You can override the prompt per-agent if you want a different shape of memory.

## Configuration

```json
{
  "memory": {
    "idleMs": 60000,
    "dreaming": {
      "schedule": "0 4 * * *"
    }
  }
}
```

- `idleMs` — how long after a session goes idle before `memory-logger` runs (default 60s)
- `dreaming.schedule` — cron expression for the daily consolidation (default `0 4 * * *`)

Both fields are restart-required because the memory plugin reads them once at factory time.

## The naming

"Dreaming" is not a metaphor for "occasional weird outputs." It's the daily consolidation step where short-term fragments become long-term memory — modeled (loosely) on how sleep consolidates episodic memory in mammals. The fact that it happens at 4am is just so the agent isn't doing it while you're trying to work with it.
